A partir de los requisitos de información definidos en la \autoref{sec:InformationRequirements}, se ha elaborado el modelo conceptual de datos del sistema. Para mejorar su legibilidad, el modelo se presenta por módulos funcionales y no como un único diagrama global excesivamente denso.

Los diagramas de esta sección abstraen el modelo físico de base de datos: se mantienen las entidades, atributos relevantes y relaciones principales, pero se omiten tipos SQL, claves técnicas y restricciones de implementación. Esta separación permite que el modelo conceptual explique el dominio del problema, mientras que el diseño físico detallado se desarrolla posteriormente en la arquitectura de almacenamiento.
