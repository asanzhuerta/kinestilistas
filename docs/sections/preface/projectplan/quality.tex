El aseguramiento de calidad del proyecto se ha planteado como un conjunto de actividades orientadas a garantizar que el sistema desarrollado sea comprensible, mantenible, funcional y coherente con los requisitos definidos. Estas actividades se han aplicado tanto al software como a la documentación generada.

\subsection{Objetivos de calidad}

Los principales objetivos de calidad considerados durante el desarrollo han sido los siguientes:

\begin{itemize}
    \item Garantizar que el sistema responde a las necesidades principales identificadas durante la captura de requisitos.
    \item Mantener una estructura de código clara y preparada para futuras ampliaciones.
    \item Asegurar que la autenticación, la gestión de usuarios y el control de roles funcionan correctamente.
    \item Favorecer la mantenibilidad mediante una organización modular del código y de la documentación.
    \item Comprobar que la aplicación puede ejecutarse en el entorno local definido.
    \item Revisar que la memoria no contenga texto de plantilla, marcadores pendientes, figuras sin referencia o inconsistencias entre capítulos.
\end{itemize}

\subsection{Prácticas aplicadas}

Durante el desarrollo se han aplicado diferentes prácticas de calidad:

\begin{itemize}
    \item \textbf{Revisión incremental del código:} los cambios se han incorporado de forma progresiva, revisando su efecto sobre el funcionamiento general del sistema.
    \item \textbf{Validación manual de flujos principales:} se han comprobado los procesos básicos de acceso, autenticación, navegación y administración inicial.
    \item \textbf{Separación de responsabilidades:} la aplicación se ha organizado en componentes, rutas, lógica de servidor y acceso a datos.
    \item \textbf{Uso de control de versiones:} se ha mantenido un historial de cambios que permite recuperar estados anteriores y revisar la evolución del proyecto.
    \item \textbf{Documentación técnica:} se han documentado los pasos necesarios para preparar el entorno y continuar el desarrollo del sistema.
    \item \textbf{Revisión de la memoria:} se ha comprobado la coherencia entre objetivos, requisitos, diseño, construcción y pruebas.
\end{itemize}

\subsection{Verificación y validación}

La verificación se ha centrado en comprobar que los artefactos generados se ajustan a lo definido en las fases anteriores del proyecto. En este sentido, se han revisado los requisitos, los modelos, la estructura del código y las pruebas realizadas.

La validación se ha orientado a comprobar que el sistema resulta adecuado para el problema planteado. Para ello, se ha contrastado el funcionamiento de la aplicación con los objetivos del proyecto y con las necesidades detectadas en la actividad del distribuidor.

\subsection{Criterios de aceptación}

Una funcionalidad se considera aceptada cuando cumple los siguientes criterios:

\begin{itemize}
    \item Está alineada con un requisito o necesidad identificada en el análisis.
    \item Puede ejecutarse correctamente en el entorno local del proyecto.
    \item No impide el funcionamiento de otros flujos previamente validados.
    \item Mantiene una estructura de código coherente con la organización general del sistema.
    \item Está documentada, al menos de forma básica, en la memoria o en los anexos correspondientes.
\end{itemize}

\subsection{Acciones correctoras}

Cuando durante el desarrollo se ha detectado un error, inconsistencia o desviación respecto al alcance previsto, se han aplicado acciones correctoras. Estas acciones han consistido principalmente en revisar el código afectado, ajustar la documentación, reducir el alcance de funcionalidades no esenciales o replantear decisiones técnicas para mantener la viabilidad del proyecto.

De este modo, el aseguramiento de calidad no se ha limitado a una fase final, sino que se ha aplicado de forma continua durante el desarrollo del Trabajo Fin de Grado.
