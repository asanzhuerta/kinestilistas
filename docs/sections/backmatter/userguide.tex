Las instrucciones de uso del software se detallan a continuación. Este manual se dirige al usuario final y describe únicamente las funcionalidades que forman parte del alcance operativo actual de la aplicación.

\section{Introducción}

KinEstilistas es una aplicación web para coordinar la relación entre un distribuidor de producto capilar profesional, su equipo comercial y sus clientes salón. En la versión actual el sistema cubre de forma efectiva la gestión de acceso y usuarios del módulo \texttt{M1}, la operativa comercial y de clientes del módulo \texttt{M2}, el catálogo del módulo \texttt{M3}, el flujo de pedidos, reparto y cobro mínimo del módulo \texttt{M4}, el seguimiento técnico interno del salón del módulo \texttt{M5}, las comunicaciones internas y multicanal, promociones, formaciones y descuentos del módulo \texttt{M6}, y la administración transversal de auditoría, avisos configurables y base técnica de integraciones del módulo \texttt{M7}.

La aplicación diferencia tres perfiles de uso:

\begin{itemize}
    \item \texttt{admin}
    \item \texttt{commercial}
    \item \texttt{client}
\end{itemize}

\section{Instalación}

Para el uso habitual del sistema desplegado solo se requiere:

\begin{itemize}
    \item un navegador moderno compatible,
    \item credenciales válidas o una solicitud de alta aprobada.
\end{itemize}

El sistema redirige automáticamente a una página informativa si detecta un navegador por debajo del soporte mínimo definido en el proyecto.

Si se desea ejecutar la aplicación en local para evaluación, deben seguirse los pasos técnicos descritos en la guía de desarrollador: levantar PostgreSQL con Docker, configurar variables de entorno, ejecutar migraciones e iniciar \texttt{Next.js}.

\section{Uso del sistema}

\subsection{Acceso inicial}

\begin{enumerate}
    \item El usuario accede a \texttt{/login} si ya dispone de cuenta.
    \item Si todavía no está dado de alta, utiliza \texttt{/register} para enviar una solicitud.
    \item Tras autenticarse correctamente, el sistema redirige al panel correspondiente según su rol.
\end{enumerate}

En el flujo de autenticación, el sistema distingue entre credenciales incorrectas y bloqueos temporales por demasiados intentos, de modo que el usuario recibe un mensaje específico cuando el acceso queda temporalmente limitado.

Una vez dentro de la aplicación, cada rol dispone de una barra lateral izquierda con las opciones disponibles para su perfil. En escritorio la barra se muestra como panel lateral, identifica el rol activo mediante los textos \textit{PROFESIONAL}, \textit{COMERCIAL} o \textit{ADMINISTRADOR}, e incorpora el logo de la aplicación. La barra puede plegarse para dejar solo los iconos y sustituye a la antigua navegación inferior. En móvil se abre desde el botón \textit{Menu}, se cierra al elegir una opción o al pulsar fuera del panel, y mantiene las mismas rutas principales que la versión de escritorio.

\subsection{Uso como administrador}

El panel de administración concentra actualmente cinco áreas principales:

\begin{itemize}
    \item \textbf{Gestión de usuarios}: revisión de solicitudes, alta manual de cuentas, consulta de usuarios y cambio de estado o rol.
    \item \textbf{Gestión de clientes}: consulta de clientes y asignaciones entre clientes y comerciales.
    \item \textbf{Gestión de catálogo}: administración de categorías, líneas, subcategorías, productos, recursos de apoyo, cartas de color y referencias cromáticas.
    \item \textbf{Gestión global de pedidos}: consulta del historial global y detalle completo de cada pedido.
    \item \textbf{Auditoría y configuración}: consulta de accesos y acciones administrativas, exportación \texttt{CSV} de trazabilidad y configuración de canales de avisos automáticos.
\end{itemize}

Las operaciones más habituales para este rol son:

\begin{enumerate}
    \item revisar solicitudes de registro pendientes;
    \item aprobar, denegar, activar o bloquear cuentas;
    \item crear usuarios desde administración;
    \item acceder al detalle de cliente para revisar sus datos;
    \item asignar o reasignar clientes a un comercial;
    \item mantener la estructura del catálogo y sus productos asociados;
    \item administrar cartas de color y referencias visuales por línea comercial;
    \item consultar la vista de auditoría en \texttt{/admin/audit}, incluyendo accesos recientes, acciones administrativas, resumen de sesiones activas y exportación \texttt{CSV} de registros de trazabilidad;
    \item configurar desde \texttt{/admin/settings} los canales adicionales de avisos automáticos;
    \item revisar el estado completo de un pedido y su trazabilidad de reparto y cobro.
\end{enumerate}

\subsection{Uso como comercial}

El panel comercial muestra una tarjeta principal de ruta diaria y varios accesos operativos de los módulos \texttt{M2}, \texttt{M3} y \texttt{M4}.

Los flujos disponibles son:

\begin{enumerate}
    \item consultar la vista previa de la ruta del día, con clientes mapeados, tiempos aproximados y margen restante;
    \item acceder al listado de clientes asignados;
    \item abrir la ficha de un cliente para consultar su información operativa;
    \item consultar, crear y actualizar visitas comerciales;
    \item revisar desde avisos las visitas aplazadas automáticamente y abrir su reubicación cuando sea necesario;
    \item ajustar la jornada base, los puntos de salida y la duración estimada de visitas en \texttt{/commercials/settings};
    \item abrir la página \texttt{/commercials/routes} para visualizar la ruta en una vista dedicada;
    \item acceder al catálogo de productos activos con filtros por categoría, línea y subcategoría;
    \item consultar fichas de producto con información técnica, recursos asociados y vinculos a coloración;
    \item explorar cartas de color y localizar tonos concretos dentro de cada línea comercial;
    \item preparar borradores de pedido y confirmarlos para clientes asignados;
    \item consultar el listado de pedidos del área comercial;
    \item gestionar visitas de reparto, vincular pedidos confirmados y completar la entrega con validación QR;
    \item registrar el cobro mínimo de pedidos ya entregados desde el detalle del pedido comercial.
\end{enumerate}

En el detalle de una visita de reparto, la confirmación de entrega se apoya en un flujo específico de QR. El comercial puede abrir el escaner, utilizar la cámara del dispositivo para capturar el código del paquete o introducir el valor manualmente si la lectura automática falla. Solo cuando el sistema dispone de todos los QR requeridos para los pedidos vinculados habilita la confirmación final de la entrega.

La información de ruta se apoya en la geolocalización del dispositivo cuando está disponible. Si el navegador no concede permiso, el sistema intenta utilizar el punto de salida guardado para el comercial.

\subsection{Uso como cliente}

En la versión actual el área cliente se centra en seis capacidades reales:

\begin{itemize}
    \item consultar la tarjeta de estimación de reparto del día;
    \item consultar el catálogo de productos activos;
    \item explorar la sección de coloración y sus cartas disponibles;
    \item gestionar su propio historial de pedidos;
    \item gestionar fichas técnicas internas de clientes del salón y registrar sus servicios;
    \item consultar y editar su perfil.
\end{itemize}

La tarjeta de reparto indica, cuando existe una visita de entrega planificada:

\begin{itemize}
    \item hora aproximada de llegada,
    \item orden previsto dentro de la ruta,
    \item franja de visita registrada,
    \item comercial asignado,
    \item estado de la previsión.
\end{itemize}

Si no existe reparto para ese día, o si la visita ha quedado fuera de la franja horaria permitida, el sistema lo comunica expresamente.

Desde \texttt{/clients/orders}, el cliente puede consultar pedidos en distintos estados, abrir su detalle y seguir la trazabilidad mínima de reparto y cobro. Desde el catálogo, puede navegar por productos activos, aplicar filtros y abrir la ficha completa de cada referencia. Esa ficha reúne información técnica, recursos de apoyo y, cuando corresponde, accesos directos a cartas de color relacionadas.

Desde la pantalla de avisos, clientes y comerciales pueden activar las notificaciones \textit{Push PWA} si el navegador lo permite y la aplicación está correctamente configurada. Al activar este canal, las comunicaciones seleccionadas por administración pueden llegar también como notificación del sistema en el dispositivo. El canal interno de avisos se mantiene siempre disponible aunque el correo o la notificación push no estén configurados.

En \texttt{/profile}, el sistema muestra el rango comercial asignado al salón: \textit{Plata}, \textit{Oro} o \textit{Platino}. El rango \textit{Plata} actúa como nivel base para todos los clientes profesionales que todavía no tengan una categoría superior asignada. Este badge resume las ventajas asociadas al segmento y enlaza con las promociones activas disponibles para el cliente, mientras que el panel inicial de cliente prioriza la información operativa de reparto.

Desde \texttt{/clients/salon-clients}, el cliente profesional puede crear fichas de clientes finales del salón, revisar su historial técnico y abrir el detalle de cada una. En dicho detalle puede actualizar la ficha base, registrar nuevos servicios con fecha, tipo de tratamiento, resumen del resultado, fórmula, notas técnicas y productos utilizados, seleccionar la tonalidad concreta cuando un tinte o producto de coloración enlaza referencias cromáticas, subir varias imágenes del resultado final, guardar plantillas reutilizables a partir del formulario actual, aplicar esas plantillas para acelerar nuevos registros, editar o eliminar servicios ya existentes y filtrar el historial por texto, tipo o fecha. Quedan además visibles las sugerencias de producto derivadas del historial acumulado y, para cada servicio, un borrador técnico editable de correo que puede copiarse o abrirse en el gestor de correo del dispositivo.

\subsection{Perfil y datos personales}

Todos los roles pueden acceder a \texttt{/profile} para editar sus datos personales. Desde esta pantalla es posible:

\begin{itemize}
    \item modificar nombre, correo, empresa y teléfono;
    \item subir o reemplazar la imagen de perfil, que también se muestra en la navegación lateral;
    \item cambiar la contraseña cumpliendo las reglas de validación;
    \item en el caso del cliente, editar la información de su ficha y confirmar la ubicación exacta del establecimiento sobre mapa.
\end{itemize}

En escritorio, la vista de perfil se organiza como un panel compacto para consultar de un vistazo la identidad, datos principales, contraseña, acciones y, cuando aplica, la ficha profesional del cliente. En dispositivos pequeños se conserva una composición vertical para priorizar la lectura y la edición segura de cada bloque.

\subsection{Observaciones sobre el alcance actual}

\textcolor{red}{Aunque la navegación ya anticipa futuros módulos del sistema, no todas las tarjetas visibles representan funcionalidades completas en esta versión.} En particular:

\begin{itemize}
    \item la previsualización de ruta existe tanto en el panel comercial principal como en una página independiente de rutas;
    \item el catálogo y la coloración forman parte del alcance operativo de comerciales y clientes;
    \item el flujo de pedidos, reparto y cobro mínimo ya está operativo;
    \item el módulo de salón ya dispone de un bloque operativo de fichas técnicas, historial con mantenimiento posterior, tonos e imágenes finales por servicio, plantillas reutilizables y borradores técnicos de correo, \textcolor{red}{aunque el envío real y otras automatizaciones siguen pendientes};
    \item promociones, formaciones, avisos internos, correo SMTP, Web Push PWA y descuentos promocionales sobre pedidos forman parte del alcance operativo actual; \textcolor{red}{quedan para iteraciones futuras \textit{WhatsApp}, otros canales externos y automatizaciones más avanzadas.}
\end{itemize}
